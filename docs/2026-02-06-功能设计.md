# NanoMind 功能设计文档

> 详细描述每个功能模块的设计，包含后端 API 和前端功能。供从零重构时参考。

---

## 一、后端 API 设计

### 1.1 API 总览

所有 API 以 `/api` 为前缀，返回 JSON 格式。

| 方法 | 路径 | 功能 | 请求体 | 响应体 |
|------|------|------|--------|--------|
| GET | `/api/files` | 获取文件树 | - | `FileTreeItem[]` |
| GET | `/api/files/{path}` | 读取文件内容 | - | `FileContentResponse` |
| PUT | `/api/files/{path}` | 保存文件内容 | `{ content }` | `{ success, path }` |
| POST | `/api/files` | 创建文件/目录 | `CreateRequest` | `{ success, path }` |
| DELETE | `/api/files/{path}` | 删除文件/目录 | - | `{ success }` |
| PATCH | `/api/files/{path}` | 重命名文件/目录 | `{ newName }` | `{ success, path }` |
| GET | `/api/search?q={query}` | 全文搜索 | - | `SearchResult[]` |

### 1.2 数据结构

#### FileTreeItem

```
{
  name: string        // 显示名称（文件不含 .md 扩展名）
  path: string        // 相对路径（文件包含 .md 扩展名）
  type: "file" | "directory"
  children?: FileTreeItem[]  // 仅目录有此字段
}
```

#### FileContentResponse

```
{
  content: string     // 文件的原始 Markdown 文本内容
  path: string        // 文件相对路径
}
```

#### CreateRequest

```
{
  path: string        // 父目录路径（空字符串表示根目录）
  type: "file" | "directory"
  name: string        // 新建项的名称（不含扩展名，后端自动为 file 追加 .md）
}
```

#### RenameRequest

```
{
  newName: string     // 新名称（不含扩展名，后端自动保留原扩展名）
}
```

#### SearchResult

```
{
  path: string        // 文件相对路径
  name: string        // 文件名（不含 .md 扩展名）
  matches: string[]   // 匹配行摘要（最多 3 条，每条最多 100 字符）
}
```

#### 错误响应

```
{
  error: string       // 错误描述
}
```

### 1.3 各 API 详细设计

---

#### B-01：获取文件树

**`GET /api/files`**

**功能描述**：
递归扫描笔记存储目录，返回完整的文件/目录树形结构。

**业务规则**：
1. 递归遍历所有子目录
2. 过滤规则（以下内容不出现在结果中）：
   - 以 `.` 开头的文件和目录（隐藏文件）
   - `node_modules` 目录
   - 非 `.md` 扩展名的文件
3. 排序规则：
   - 目录排在文件前面
   - 同类型按名称字母升序排列
4. 文件的 `name` 字段不含 `.md` 扩展名
5. 文件的 `path` 字段保留 `.md` 扩展名

**响应示例**：
```
[
  {
    "name": "工作",
    "path": "工作",
    "type": "directory",
    "children": [
      {
        "name": "会议记录",
        "path": "工作/会议记录.md",
        "type": "file"
      }
    ]
  },
  {
    "name": "个人笔记",
    "path": "个人笔记.md",
    "type": "file"
  }
]
```

**边界情况**：
- 笔记目录不存在时返回空数组 `[]`
- 笔记目录为空时返回空数组 `[]`
- 永远返回数组（不返回 null）

---

#### B-02：读取文件内容

**`GET /api/files/{path}`**

**功能描述**：
按路径读取指定 `.md` 文件的原始文本内容。

**业务规则**：
1. `{path}` 为文件相对路径（如 `工作/会议记录.md`）
2. 路径参数需要进行 URL 解码（处理中文、空格等特殊字符）
3. 必须校验解码后的完整路径在笔记目录范围内（防目录遍历）
4. 返回文件原始文本内容（Markdown 格式）

**错误处理**：
| 场景 | 状态码 | 错误信息 |
|------|--------|---------|
| 路径为空 | 400 | File path required |
| URL 解码失败 | 400 | Invalid file path |
| 路径超出笔记目录 | 403 | Access denied |
| 文件不存在 | 404 | File not found |

---

#### B-03：保存文件内容

**`PUT /api/files/{path}`**

**功能描述**：
将内容写入指定路径的文件。如果文件所在目录不存在则自动创建。

**业务规则**：
1. 请求体包含 `content` 字段（Markdown 文本）
2. 路径参数需要 URL 解码
3. 写入前自动创建不存在的父目录
4. 文件权限：0644
5. 目录权限：0755

**错误处理**：
| 场景 | 状态码 | 错误信息 |
|------|--------|---------|
| 路径为空 | 400 | File path required |
| 请求体解析失败 | 400 | Invalid request |
| 路径超出笔记目录 | 403 | Access denied |
| 写入失败 | 500 | Failed to save file |

---

#### B-04：创建文件/目录

**`POST /api/files`**

**功能描述**：
在指定父目录下创建新文件或新目录。

**业务规则**：
1. `name` 不能为空
2. 创建文件时：
   - 自动追加 `.md` 扩展名
   - 初始内容为 `# {name}\n\n`（以文件名作为一级标题）
   - 文件权限：0644
3. 创建目录时：
   - 目录权限：0755
4. 父路径为空字符串时表示在根目录下创建
5. 目标已存在时返回冲突错误

**错误处理**：
| 场景 | 状态码 | 错误信息 |
|------|--------|---------|
| 名称为空 | 400 | Name required |
| 路径超出笔记目录 | 403 | Access denied |
| 同名文件/目录已存在 | 409 | File or directory already exists |
| 创建失败 | 500 | Failed to create file/directory |

**成功响应**：返回新创建项的相对路径。

---

#### B-05：删除文件/目录

**`DELETE /api/files/{path}`**

**功能描述**：
删除指定路径的文件或目录。

**业务规则**：
1. 如果是目录，递归删除（包含所有子文件和子目录）
2. 如果是文件，直接删除
3. 路径参数需要 URL 解码

**错误处理**：
| 场景 | 状态码 | 错误信息 |
|------|--------|---------|
| 路径为空 | 400 | File path required |
| 路径超出笔记目录 | 403 | Access denied |
| 文件/目录不存在 | 404 | File not found |
| 删除失败 | 500 | Failed to delete |

---

#### B-06：重命名文件/目录

**`PATCH /api/files/{path}`**

**功能描述**：
重命名指定路径的文件或目录。

**业务规则**：
1. `newName` 不能为空
2. 自动保留原文件的扩展名（即只改名字不改扩展名）
3. 重命名后的文件仍在同一目录下
4. 目标名称已存在时返回冲突错误
5. 成功后返回新的相对路径

**错误处理**：
| 场景 | 状态码 | 错误信息 |
|------|--------|---------|
| 路径为空 | 400 | File path required |
| 新名称为空 | 400 | New name required |
| 路径超出笔记目录 | 403 | Access denied |
| 原文件不存在 | 404 | File not found |
| 目标已存在 | 409 | Target already exists |
| 重命名失败 | 500 | Failed to rename |

---

#### B-07：全文搜索

**`GET /api/search?q={query}`**

**功能描述**：
在所有 `.md` 文件中搜索包含关键词的文件。

**业务规则**：
1. 最小查询长度：2 个字符（少于 2 个字符返回空数组）
2. 搜索方式：大小写不敏感的子串匹配
3. 搜索范围：所有 `.md` 文件的全文内容
4. 结果限制：最多返回 20 条结果
5. 每条结果包含匹配行摘要：
   - 最多 3 条匹配行
   - 每行最多 100 个字符（超长截断）
   - 去除行首尾空白
6. `name` 字段不含 `.md` 扩展名

**性能考虑**：
- 当前为简单的文件遍历 + 内容读取，适用于小型笔记库
- 大型笔记库可考虑引入索引机制

---

#### B-08/B-09：静态资源服务与 SPA 回退

**功能描述**：

1. **静态资源服务**：将前端构建产物（HTML/CSS/JS/图片/字体等）嵌入到后端二进制中，通过 HTTP 提供服务
2. **SPA 路由回退**：
   - 对于没有文件扩展名的路径（如 `/some/page`），返回 `index.html`（支持前端路由）
   - 对于有文件扩展名但不存在的路径（如 `/missing.js`），返回 404

**支持的静态资源类型**：
| 扩展名 | 用途 |
|--------|------|
| `.html` | HTML 页面 |
| `.css` | 样式表 |
| `.js` | JavaScript |
| `.json` | JSON 数据 |
| `.png`, `.jpg`, `.svg`, `.ico` | 图片 |
| `.woff`, `.woff2`, `.ttf` | 字体 |

---

#### B-10：路径安全校验

**所有涉及路径操作的 API 必须执行以下校验**：

1. 路径参数进行 URL 解码
2. 仅当路径包含 `%` 时才解码（避免重复解码）
3. 解码失败返回 400
4. 拼接完整路径后验证是否在笔记目录范围内（前缀校验）
5. 不在范围内返回 403

## 二、前端功能设计

### 2.1 页面布局

整个应用为单页面应用 (SPA)，布局分为四个区域：

```
┌──────────┬──────────────────────────┬──────────┐
│          │        顶部栏 (Header)     │          │
│  文件树   ├──────────────────────────┤  目录导航  │
│  侧边栏   │                          │ (TOC)    │
│  288px   │      主内容区 (Editor)      │  224px   │
│          │       弹性宽度             │          │
│          │                          │          │
└──────────┴──────────────────────────┴──────────┘
```

- 整体最大宽度：1700px，居中显示
- 超过最大宽度时两侧出现背景色
- 全屏高度布局

### 2.5 F-04：Markdown 双向转换

**加载流程（Markdown → HTML）**：
1. API 返回原始 Markdown 文本
2. 转换规则：
   - 使用 GFM (GitHub Flavored Markdown) 语法
   - 不自动将换行转为 `<br>`（`breaks: false`）
   - 任务列表特殊处理：
     - 预处理：提取连续的 `- [ ]` / `- [x]` 行，替换为占位符
     - 使用 Markdown 解析器处理其余内容
     - 后处理：将占位符替换为带 `data-type="taskList"` 和 `data-checked` 属性的 HTML

**保存流程（HTML → Markdown）**：
1. 遍历 DOM 节点树，按标签类型生成对应 Markdown 语法
2. 转换映射：
   - `<h1>` → `# `，`<h2>` → `## `，`<h3>` → `### `
   - `<strong>/<b>` → `**text**`
   - `<em>/<i>` → `*text*`
   - `<u>` → `<u>text</u>`（保留 HTML 标签）
   - `<s>/<del>` → `~~text~~`
   - `<code>`（非 pre 内）→ `` `text` ``
   - `<pre><code>` → ` ```\ncode\n``` `
   - `<blockquote>` → `> text`
   - `<ul>` → `- item`
   - `<ol>` → `1. item`
   - `<ul data-type="taskList">` → `- [ ] item` / `- [x] item`
   - `<a>` → `[text](href)`
   - `<hr>` → `---`
   - `<table>` → Markdown 表格格式
   - `<br>` → `\n`

### 2.6 F-05：全局搜索

**触发方式**：`Ctrl/Cmd+K` 或点击侧边栏搜索框

**搜索弹窗设计**：

#### 搜索输入
- 左侧搜索图标
- 搜索输入框
- 右侧搜索模式切换按钮（文件图标 / 全文图标）
- 输入防抖：200ms
- 最小查询长度：2 字符

#### 搜索模式
- **文件名搜索**（默认）：在本地文件索引中过滤，匹配文件名或路径
- **全文搜索**：调用后端搜索 API，合并文件名匹配结果
- 切换快捷键：`Ctrl/Cmd+D`（在搜索弹窗内）

#### 全文搜索合并逻辑
1. 先执行文件名过滤（最多 20 条）
2. 再执行 API 全文搜索
3. 合并结果：
   - 同路径文件去重
   - 已存在的文件名结果补充匹配摘要
   - 不存在的全文结果追加到末尾

#### 最近文件
- 搜索弹窗打开时显示最近打开的文件（最多 5 个）
- 以标签（pill）样式水平排列
- 可点击直接打开

#### 搜索结果列表
- 每条结果显示：文件名 + 路径 + 匹配摘要（如有）
- 可滚动，最大高度 256px

### 2.7 F-06：目录导航 (TOC)

**位置**：右侧，固定宽度 224px (w-56)

**显示条件**：仅在屏幕宽度 >= xl 断点时显示

**组成部分**：

#### 标题
- "On this page" + 列表图标
- 大写字母、小字号

#### 目录列表
- 从当前文档的 HTML 内容中提取 H1、H2、H3 标题
- 每个标题项生成唯一 ID（slugify 处理）
- 按标题层级缩进：
  - H1：左边距 12px
  - H2：左边距 24px
  - H3：左边距 36px

#### 交互行为

  - 高亮逻辑：遍历所有标题元素，找到最后一个 `offsetTop - 12 <= scrollTop` 的标题
- **空状态**：无标题时显示"暂无目录"

#### 标题 ID 注入
- 在文档渲染后，为 DOM 中的 H1/H2/H3 元素动态设置 ID
- ID 与 TOC 列表中的项对应

### 2.16 F-15：Markdown 粘贴

**功能**：在编辑器中粘贴纯文本时，如果检测到 Markdown 格式，自动转换为富文本。

**检测规则**（满足任一即认为是 Markdown）：
- 标题：`# `、`## ` 等
- 列表：`* `、`- `、`1. `
- 加粗：`**text**`
- 斜体：`*text*`
- 行内代码：`` `code` ``
- 代码块：` ``` `
- 引用：`> `
- 链接：`[text](url)`
- 表格：`|...|`

**行为**：
- 如果剪贴板同时包含 HTML 格式，优先使用 HTML（不触发 Markdown 转换）
- 仅当只有纯文本且检测为 Markdown 时才转换
- 转换后替换当前选区

### 2.18 排版样式（Typography）

**基础设置**：
- 字体族：Inter, 系统字体栈
- 根字号：120%（约 19.2px）
- 行高：1.6

**内容排版**：
- 基础字号：15px
- 段落间距：8px (my-2)
- H1：1.5rem，半粗，字间距紧凑，上边距 0，下边距 12px
- H2：1.25rem，半粗，上边距 20px，下边距 8px
- H3：1.125rem，半粗，上边距 16px，下边距 8px

- 列表：左缩进 20px
- 行内代码：单字号，背景色，小圆角
- 代码块：暗色背景 (#1e1e1e)，13px 字号，圆角
- 表格：全宽，边框，表头背景色
- 链接：主题色，半粗，悬停下划线
- 分隔线：边框色
