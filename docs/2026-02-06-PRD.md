# NanoMind 产品需求文档 (PRD)

> 本文档从现有项目中提炼出完整的产品需求，用于从零重构。不涉及具体技术选型。

---

## 一、产品定位

### 1.1 产品名称

NanoMind

### 1.2 一句话描述

一个自托管的轻量级 Markdown 笔记应用，提供 Web 端富文本编辑体验，以单个可执行文件部署。

### 1.3 核心理念

- **极简部署**：单个二进制文件，零外部依赖（无需数据库、无需 Docker），下载即用
- **本地存储**：所有笔记以 Markdown 文件形式存储在本地文件系统，数据完全归用户所有
- **富文本编辑**：所见即所得（WYSIWYG）的编辑体验，同时底层保持 Markdown 格式的可移植性
- **自托管优先**：面向个人或小团队，部署在自己的服务器或本地机器上

### 1.4 目标用户

| 用户类型 | 使用场景 |
|---------|---------|
| 个人开发者 | 在服务器上部署，随时通过浏览器记录技术笔记、日志 |
| 知识工作者 | 管理个人知识库，整理分类笔记 |
| 小团队 | 在内部服务器部署，共享文档和知识 |
| NAS 用户 | 在家庭 NAS 上部署，作为轻量级笔记服务 |

### 1.5 产品边界（做什么/不做什么）

| 做 | 不做 |
|---|------|
| Markdown 文档的创建、编辑、管理 | 多用户系统/权限管理 |
| 文件/目录树形管理 | 实时协作编辑 |
| 全文搜索 | 文件版本历史 |
| 多主题、暗色模式 | 图片/附件上传 |
| 单二进制部署 | 数据库存储 |
| 富文本编辑器 | 移动端原生 App |
| 代码块语法高亮 | 用户认证/登录系统 |
| 目录导航（TOC） | 导出为 PDF/Word |

## 二、核心概念与数据模型

### 2.1 核心实体

#### 文档 (Memo/Note)

- 本质是一个 `.md` 文件
- 存储格式：标准 Markdown
- 显示格式：富文本（HTML）
- 编辑时在 HTML 富文本与 Markdown 之间双向转换
- 新建文档时自动以文档名称生成一级标题作为初始内容（如 `# 文档名\n\n`）
- 文件扩展名固定为 `.md`，前端显示时自动附加/隐藏扩展名

#### 目录 (Directory)

- 用于组织文档的文件夹
- 支持多级嵌套
- 自动过滤隐藏文件（以 `.` 开头的文件/目录）
- 自动过滤非 `.md` 文件

#### 文件树 (File Tree)

- 整个笔记库的树形结构表示
- 排序规则：目录优先，目录和文件各自按名称字母升序排列
- 递归构建，无深度限制

### 2.2 数据存储

- **存储介质**：本地文件系统
- **存储目录**：可配置（默认 `~/.nanomind/memos`）
- **文件编码**：UTF-8
- **无数据库**：完全依赖文件系统，不使用任何数据库

### 2.3 数据流

```
用户编辑（富文本 HTML）
      ↓
  HTML → Markdown 转换
      ↓
  通过 API 保存
      ↓
  写入 .md 文件到文件系统
```

```
用户打开文档
      ↓
  API 读取 .md 文件内容
      ↓
  Markdown → HTML 转换
      ↓
  富文本编辑器渲染
```

## 三、功能清单总览

### 3.1 后端功能

| 编号 | 功能 | 描述 |
|------|------|------|
| B-01 | 文件树获取 | 递归扫描笔记目录，返回树形结构 |
| B-02 | 文件内容读取 | 按路径读取 .md 文件内容 |
| B-03 | 文件内容保存 | 按路径写入 .md 文件内容 |
| B-04 | 文件/目录创建 | 新建 .md 文件或目录 |
| B-05 | 文件/目录删除 | 删除文件或递归删除目录 |
| B-06 | 文件/目录重命名 | 重命名文件或目录 |
| B-07 | 全文搜索 | 在所有 .md 文件中搜索关键词 |
| B-08 | 静态资源服务 | 提供前端 SPA 静态文件服务 |
| B-09 | SPA 路由回退 | 非 API、非静态文件的请求回退到 index.html |
| B-10 | 路径安全校验 | 防止目录遍历攻击 |

### 3.2 前端功能

| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 文件树侧边栏 | 树形展示文件结构，支持展开/折叠 |
| F-02 | 富文本编辑器 | 所见即所得编辑，支持丰富的文本格式 |
| F-03 | 阅读/编辑模式切换 | 默认阅读模式，手动进入编辑模式 |
| F-04 | Markdown 双向转换 | 加载时 MD→HTML，保存时 HTML→MD |
| F-05 | 全局搜索 | 弹窗式搜索，支持文件名和全文搜索 |
| F-06 | 目录导航（TOC） | 从文档标题自动生成目录，点击跳转 |
| F-07 | 主题系统 | 多套主题 + 暗色模式 |
| F-08 | 快捷键系统 | 全局快捷键和编辑器快捷键 |
| F-09 | 文件操作（CRUD） | 新建/重命名/删除文件和目录 |
| F-10 | 最近文件 | 记录最近打开的文件，快速访问 |
| F-11 | 编辑器工具栏 | 顶部固定工具栏，提供所有格式化操作 |
| F-12 | 浮动工具栏 | 选中文本时弹出快捷格式化工具栏 |
| F-13 | 未保存变更提示 | 跟踪编辑变更，防止意外丢失 |
| F-14 | 面包屑导航 | 顶部栏显示当前文件路径 |
| F-15 | Markdown 粘贴 | 粘贴 Markdown 文本自动转换为富文本 |

## 五、非功能性需求

### 5.1 性能

- 文件树扫描应在 1000 个文件以内秒级响应
- 搜索结果应在 200ms 内返回（小型笔记库）
- 前端首屏加载应在 2 秒内完成

### 5.2 安全

- 所有文件操作必须限制在配置的笔记目录范围内（防目录遍历）
- 路径参数需要 URL 解码处理
- 无需认证（自托管场景，由部署者自行保障网络安全）

### 5.3 可用性

- 响应式布局，适配桌面端浏览器
- 支持暗色/亮色模式
- 所有交互有明确的视觉反馈
- 支持键盘快捷键操作

### 5.4 部署

- 单二进制文件，零运行时依赖
- 支持跨平台（Linux amd64/arm64、macOS Intel/Apple Silicon、Windows）
- 前端静态资源嵌入到二进制中
- 启动后即可使用，无需额外配置
