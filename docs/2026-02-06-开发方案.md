# Bugfix: Markdown 粘贴无法渲染

## 问题

直接将 Markdown 格式文本粘贴到 Tiptap 编辑器时，内容被当作纯文本处理，Markdown 特殊字符被转义（每行前加 `\`），无法渲染为富文本。

## 原因

Tiptap 默认仅处理 HTML 粘贴内容。当剪贴板只有纯文本（即 Markdown 源码）时，Tiptap 直接将其作为纯文本节点插入，ProseMirror 的输入规则会转义 Markdown 语法字符。

## 方案

1. 在 `markdown.ts` 中新增 `looksLikeMarkdown(text)` 检测函数，通过行级模式匹配判断文本是否为 Markdown 格式
2. 在 `TiptapEditor.tsx` 的 `editorProps.handlePaste` 中拦截粘贴事件：
   - 若剪贴板包含 HTML → 交给 Tiptap 默认处理
   - 若仅有纯文本且检测为 Markdown → 调用 `markdownToHtml()` 转换后以 HTML 插入编辑器
   - 否则 → 交给 Tiptap 默认处理

## 修改文件

- `site/src/shared/lib/markdown.ts` — 新增 `looksLikeMarkdown()` 导出
- `site/src/views/editor/components/TiptapEditor.tsx` — 添加 `handlePaste` 拦截逻辑
